## üìù Documentation Guidelines

### Writing Guidelines
1. Keep documentation up-to-date
2. Use clear, descriptive names
3. Max file size: 100KB
4. Include code examples where applicable
5. Add troubleshooting sections

### Organization Rules
1. **Current** docs: Actively maintained, production-relevant
2. **Archive** docs: Historical reference, completed projects
3. **Review** quarterly: Move outdated docs to archive
4. **Index** required: Update this README when adding new docs

### File Naming Convention
```
[CATEGORY]_[TOPIC]_[TYPE].md

Examples:
- DEPLOYMENT_GUIDE.md
- API_AUTHENTICATION_GUIDE.md
- TROUBLESHOOTING_DATABASE_ERRORS.md
```
